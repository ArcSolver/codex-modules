# 모듈 규칙

## 기본 구조

`modules/<name>/`는 완전히 독립적인 npm 패키지여야 한다.

- 크로스 모듈 의존은 금지한다.
- 외부 의존성은 최소화한다.
- 한 모듈을 복사하거나 배포해도 다른 모듈 없이 동작해야 한다.

## 필수 구성

- `README.md`: install, usage, how it works, Attribution, uninstall-rollback 섹션을 포함한다.
- `package.json`: npm 패키지 메타데이터, scripts, bin 또는 exports를 명시한다.
- `src/`: TypeScript로 작성하며 Node >= 20 및 bun에서 동작해야 한다.
- `verify/`: 실행 가능한 검증 스크립트를 둔다. 검증은 모듈이 의도대로 동작함을 자동으로 증명해야 한다.

## API와 CLI

가능하면 프로그래매틱 API와 CLI를 모두 노출한다.

- API는 다른 도구에서 재사용할 수 있게 작고 명확해야 한다.
- CLI는 신규 사용자가 README만 보고 실행할 수 있어야 한다.
- CLI 옵션은 명시적이고 롤백 가능한 동작을 기본으로 설계한다.

## 안전 규칙

- 사용자의 실제 `$CODEX_HOME`은 명시적 동작 없이 수정하지 않는다.
- 모든 설치/검증 명령은 `CODEX_HOME` 오버라이드를 지원해야 한다.
- 파괴적 변경 전에는 백업을 생성한다.
- 설치한 변경은 완전한 롤백 또는 uninstall 경로를 제공해야 한다.
- 실패 시 사용자가 어떤 파일을 확인하고 어떤 명령으로 되돌릴 수 있는지 README에 적는다.

## 네이밍

- 모듈 디렉토리는 kebab-case를 사용한다.
- npm 패키지명은 `codex-<name>` 계열을 사용한다.
- 공개 CLI 명령도 패키지명과 기능을 쉽게 연결할 수 있어야 한다.
